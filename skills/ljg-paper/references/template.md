# 速读

## 一句话

{谁原来怎么想；这篇论文说应该怎么改。}

## 最值得记住

{只能带走一个判断，带走哪一个。}

## 先别误会

{这篇没有证明什么，或者不能推出什么。}

# 它在反对什么

{写旧看法。可以是领域共识、产品直觉、研究习惯，也可以是普通人容易犯的误解。}

{说清这篇论文为什么觉得旧看法不够好。不要急着介绍作者方法。}

# 它的新想法

{把作者真正押注的判断写成一句人话。}

{必要时交代方法，但只讲到读者能理解新判断为止。不要展开完整技术流程。}

# 为什么可以信一点

## 它看见了什么

{作者抓到的现象、失败模式或反常结果。}

## 它怎么确认了一下

{最小验证是什么。别展开实验流水账。}

## 我信到几分

{强 / 中 / 弱。给一句理由。数字只在会改变判断时写，并翻成人话。}

# 我该怎么更新世界模型

{把论文转成一条以后能用的判断规则。}

{推荐句式：以后看到 X，我先不急着判断 Y，而会先问 Z。}

{说明它落在哪个大模型母题里：能力边界 / 训练机制 / 推理方式 / Agent 组织 / 评测幻觉 / 产品启发。}

# 边界

{它在哪些模型、任务、规模、场景里才可能成立。}

{普通读者最容易把它误读成什么。}
