# ljg-book：读完后有一点不同

让一本书在读者身上留下一处可观察的变化。

下次遇到某个具体情境，读者会多看见一个信号、换一个问题、换一个小动作，或克制一个旧反应。四种变化只选一种，只留一个。

不要写章节摘要、读后感或审稿报告。能复述全书不算完成；能说出「因为读了这本书，下一次我会哪里不同」才算。

## Workflow Routing

| 输入 | 必读 | 输出 |
|---|---|---|
| 书名 | 查到可靠材料后读 `ReadingGuide.md` | 保存 org 笔记 |
| PDF、正文、样章、旧笔记 | 先读材料，再读 `ReadingGuide.md` | 保存 org 笔记 |
| 用户明确只要口头解释 | `ReadingGuide.md` | 不写文件，按同一路径讲 |

写 org 文件时再读 `references/template.md`。默认保存到 `~/Documents/notes/`。

文件名沿用 Denote：`{YYYYMMDDTHHMMSS}--拆书-{书名}__book.org`；时间戳用 `date +%Y%m%dT%H%M%S` 生成。

## Completion Target

每份笔记最后只落下一种变化：

- *看见*：以前忽略什么，以后会注意什么。
- *提问*：以前先问什么，以后换成什么问题。
- *行动*：以前先做什么，以后换一个小动作。
- *克制*：以前会自动做什么，以后先停一下。

用一句「变化句」检验：

> 下次当 X 出现，我先 Z，而不是 Y。

文学作品不必硬造行动。它可以让人下次多看见一点、少急着判断一次，或允许一种过去容不下的经验。

## Gotchas

- *全书中心可能太大。* 不必带走最宏大的命题；优先选择最可能改变下一个具体时刻的小东西。
- *一个框架仍可能装得太多。* 如果结果要求记住四步、四问或整张清单，继续缩，直到只剩一个触发和一个新反应。
- *可执行不等于有影响。* 文学、传记和回忆录常改变注意力或判断，不要改写成效率工具。
- *术语不是记忆锚。* 必须找到一个场景、实验、人物片段、区分或比喻，让收获有具体载体。
- *不要伪造「对我」。* 上下文有读者的真实问题就接上；没有就写可观察的普通情境，不编生活故事。
- *边界不要另起炉灶。* 只在变化容易误用时，紧接着补一句「别在什么情况下用」。
- *材料薄就少说。* 只有目录、访谈或书评时标为「初拆」；证据更薄时标为「假设版」。
- *允许没有宏大收获。* 一处更准确的注意、一个少犯的错误，也比硬造人生启示有价值。

## Quick Reference

1. 找出书在处理的一个真实困境，不把话题当问题。
2. 在内部提出两三个可能带走的变化，只选一个。
3. 为它找一个书中独有的记忆锚。
4. 写出一个触发情境和一个替代反应。
5. 其余正文只负责让这句话可信、清楚、记得住。

开头固定三行：

1. `- *问题*：...`
2. `- *洞见*：...`
3. `- *变化*：下次当 X 出现，我先 Z，而不是 Y。`

顶层章节固定为：

1. `* 它在解决什么`
2. `* 它让我看见什么`
3. `* 下一次，哪里会不同`
4. `* 资料校准`

## Examples

**分析类书**

《决策与判断》不要留下四道偏差检查。可以只留：「下次重要选择被写成损失时，我先把它改写成获得再决定；如果答案变了，就先暂停。」

**文学作品**

《悉达多》不要留下四个自省问题。可以只留：「下次急着向别人要现成答案时，我先问：这里有什么只能由我亲自经历，不能由别人替我理解？」

## Completion

生成后读回确认：

- 文件确实保存，frontmatter 完整。
- 开头只有「问题 / 洞见 / 变化」三个落点。
- 顶层只有四个规定章节，没有独立「边界」。
- 变化句只有一个触发和一个新反应，没有重新长成清单。
- 正文有一个书中独有的记忆锚。
- 只读前三行，就能回答：「因为这本书，下一次我会哪里不同？」
