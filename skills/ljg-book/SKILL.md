# ljg-book: 拆书

输入一本书，输出一个能带走的东西。

不要写读后感。不要写章节摘要。不要写审稿报告。

这份 skill 只问四件事：

1. 作者在答什么问题？
2. 他给了什么新见解？
3. 我以后遇到这个问题，怎么用它？
4. 它不能用在哪里？

宁可少，但要能吸收。别为了全面，把主线写散。

## 核心目标

读者半年后只需要记住三句话：

- 这本书处理的问题是：……
- 作者给的新东西是：……
- 我以后可以这样用：……

如果这三句话写不出来，说明还没拆到位。

## 先看材料

只给书名时，先查：

- 作者、出版社页、目录、导言或样章。
- 作者访谈或可靠书评。
- 这个问题之前的常见答案。

给了 PDF、正文或样章时，先读原文。除非用户禁止联网，仍要轻查一下基线，别只信作者自己树的靶子。

材料不够就降级：

- 有全文或足够书内材料：可以完整拆。
- 只有目录、导言、访谈、书评：写「初拆」，少下定论。
- 资料很薄：写「假设版」，不装读过。

材料等级不要放在开头抢戏。放到末尾「资料校准」里。

## 正文结构

文件头之后、第一个标题之前，先放三行：

```org
- *问题*：作者在答什么问题
- *新见解*：概念 / 方法 / 框架 / 一句话，四选一
- *用法*：以后遇到这个问题，我该怎么用它
```

正文 3 到 5 节。标题从这本书里长出来，不用固定模板。推荐顺序：

1. 问题：作者到底在处理什么麻烦。
2. 新见解：他给出的新东西是什么。
3. 用法：这个东西以后怎么用。
4. 边界：哪里不能乱用。
5. 资料校准：证据从哪里来。

图、参考系、流派地图都不是必选。只有它能让「问题 -> 新见解 -> 用法」更清楚时才画。画了也要短。

## 1. 找问题

问题不是话题。

「科技与未来」是话题。「科技演化有没有自己的方向」才是问题。

找问题时问：

- 作者为什么要写这本书？
- 他觉得大家原来哪里想错了？
- 读者翻开书前，多半怎么想？
- 这本书如果只回答一个问题，那句话是什么？

写法要具体。拿去描述同领域另一本书也成立，就太泛。

## 2. 找新见解

新见解只挑一个主件，不要堆。

它可以是：

- *概念*：作者铸的词，或重新命名的东西。
- *方法*：一组做法、检查顺序、训练动作。
- *框架*：看问题的一副眼镜。
- *一句话*：全书最能站住的判断。

用书里的词。必须出现 1-3 个这本书自己的术语、清单或区分。没有这些词，你拆的是领域，不是这本书。

不要急着判作者是不是原创。更要紧的是：这个东西让我以后怎么看、怎么做。

可以顺手写一句来源：

- 这是作者新造的。
- 这是借来的，但用到新地方。
- 这是旧框架，作者把它变成了一套动作。
- 证据不够，来源不强判。

但来源判断只是一句，不要变成主菜。

## 3. 写用法

这是最重要的一节。

把新见解翻成一个可执行动作：

- 以后遇到什么问题时用它？
- 先问哪一个问题？
- 看哪一个信号？
- 做哪一个动作？
- 避免哪一种误用？

写成这样的句子：

「以后遇到 X，不要先 Y；先用作者的 Z，看 A，再决定 B。」

用法必须能离开这本书。不能只说「我们要更有耐心」「要系统思考」这种软话。

## 4. 写边界

每个好见解都有代价。边界写短，但不能省。

问：

- 什么情况下它会误导人？
- 它会漏掉什么？
- 它默认了什么条件？
- 如果照它做，可能付什么代价？

边界不是挑刺。边界是防止把一本书当万能钥匙。

## 5. 资料校准

末尾放短短一节即可。3 到 6 条。

每条说明材料类型和链接：

- 书内证据：全文、样章、目录、导言。
- 作者材料：访谈、演讲、作者主页、出版方介绍。
- 外部基线：前人流派、争论、批评。
- 现实材料：用于验证用法或边界的案例。

资料校准服务主线，不要压过正文。

## 输出

写入：

`~/Documents/notes/{时间戳}--拆书-{书名}__book.org`

时间戳：

- `date +%Y%m%dT%H%M%S`
- `date "+%Y-%m-%d %a %H:%M"`

文件头：

```org
#+TITLE: 拆书：《{书名}》
#+SUBTITLE: {作者} | {一句话核心观点}
#+DATE: [{YYYY-MM-DD Day HH:MM}]
#+FILETAGS: :book:{领域}:
#+IDENTIFIER: {YYYYMMDDTHHMMSS}
```

正文控制在能读完、能带走的长度。默认 3 到 5 节。用户没要求深拆时，不要写百科。

## 文风

简洁，直白，像人说话。

不要夸作者。不要贬作者。不要替作者辩护。

少用：

- 作者认为
- 作者论证
- 第 N 章指出
- 在这个意义上
- 通过某某视角
- 提供了一种可能性

多写：

- 他把 X 改成 Y。
- 他给了一个检查顺序。
- 这东西以后可以这样用。
- 这里会误伤。

禁用这些旧口头禅：

- 骨架抽出来
- 精神内核就在你手里
- 走两步
- delta
- 钉死
- 这一刀
- 锋利
- 砸实
- 带走的那一件

每段读一遍。卡住就整句重写。不要靠形容词制造力量。

## 交稿前自查

交付前只查这八条：

1. 开头三行是不是一眼能懂？
2. 问题是不是具体，不是话题？
3. 新见解是不是只挑了一个主件？
4. 书里自己的词有没有出现？
5. 用法是不是能离开这本书？
6. 边界是不是说清了代价？
7. 资料有没有支撑关键判断？
8. 有没有为了全面写散主线？

最后问自己一句：读者读完能不能把一个东西吸收进来？
